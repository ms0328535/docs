Welcome to the **Hospital Management System** — a lightweight, console-based application built entirely in C that helps hospital staff register patients, maintain a doctor registry, queue appointment requests, and schedule (or undo) appointments. Every feature is backed by a carefully chosen classic data structure, so the system stays fast and predictable even as your data grows.

## Get Up and Running

Follow these steps to go from a fresh terminal to your first scheduled appointment.

<Steps>
  <Step title="Compile the source">
    Use any C99-compatible compiler. With GCC, run:

    ```bash
    gcc -std=c99 -Wall -o hospital project.c
    ```

    A successful compilation produces the `hospital` executable with no warnings.
  </Step>

  <Step title="Launch the application">
    Start the program from your terminal:

    ```bash
    ./hospital
    ```

    You will immediately see the **Hospital Simple Menu** with options 0 through 10.
  </Step>

  <Step title="Add a doctor and register a patient">
    At the menu prompt, enter **4** to add a doctor, then type a name (e.g., `Dr. Smith`). The system assigns an auto-incremented ID and inserts the record into the BST. Next, enter **1** to register a patient, provide their name (e.g., `Jane Doe`), and note the patient ID that is printed back to you.
  </Step>

  <Step title="Request and schedule an appointment">
    Enter **6** to enqueue an appointment request. Supply the patient ID, the doctor ID, and a date/time string (e.g., `2025-09-01 09:00`). The request is placed at the rear of the circular queue. Finally, enter **8** to process the next request — the system dequeues it, confirms both IDs exist, creates a scheduled appointment, and pushes the schedule ID onto the undo stack so you can reverse it with option **9** if needed.
  </Step>
</Steps>

## Feature Highlights

<CardGroup cols={2}>
  <Card title="Patient Management" icon="bed-pulse" href="/introduction#patient-management">
    Add, remove, and list patients stored in a singly linked list. Each patient receives a unique auto-incremented ID the moment they are registered.
  </Card>
  <Card title="Doctor Registry" icon="user-doctor" href="/introduction#doctor-registry">
    Doctors are organised in a Binary Search Tree keyed by ID, enabling fast look-up during scheduling. An in-order traversal always displays them in sorted order.
  </Card>
  <Card title="Appointment Queue" icon="clock-rotate-left" href="/introduction#appointment-queuing">
    Appointment requests are held in a circular FIFO queue with a capacity of 8 slots. Requests are processed one at a time in the order they were submitted.
  </Card>
  <Card title="Undo Scheduling" icon="rotate-left" href="/introduction#schedule-management-with-undo">
    Every confirmed appointment is pushed onto a LIFO stack (capacity 50). Enter **9** at any time to instantly cancel the most recently scheduled appointment.
  </Card>
</CardGroup>
